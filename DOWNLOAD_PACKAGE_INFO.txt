HOW THE OPTIMAL CALCULATOR WORKS
═════════════════════════════════

After each completed stint, the calculator:

1. Analyzes Performance
   • Average stint time
   • Consistency score
   • Recent trends

2. Calculates Requirements
   • Remaining race time
   • Remaining driver changes
   • Time needed per stint

3. Provides Recommendation
   • Weighted formula: 60% performance + 40% requirements
   • Strategic advice (extend/maintain/shorten stints)
   • Confidence level based on data quality

Example Output:
  ┌─────────────────────────────────────────────┐
  │ Recommended: 32:15                          │
  │ "Maintain current pace - on track"          │
  │ Confidence: 85%                             │
  │                                              │
  │ Performance:                                │
  │ • Average: 31:30                            │
  │ • Consistency: 88%                          │
  │ • At Target: 7/9 stints                     │

QUICK TROUBLESHOOTING
══════════════════════

Issue: npm not found
Fix: Install Node.js from nodejs.org

Issue: Port 3000 in use
Fix: Change PORT in server.js or stop conflicting app

Issue: Can't connect in browser
Fix: Ensure server is running, try http://127.0.0.1:3000

Issue: Dependencies won't install
Fix: Check internet, try npm install --force

For detailed troubleshooting → INSTALL.md


MULTI-DEVICE ACCESS
═══════════════════
